Compass is built for one type of person: someone who can build a working app but is figuring out the business side for the first time.

It's not generic startup advice. It's written for the solo founder or small team — using AI tools or writing their own code — who shipped something, has users, and now needs to know what to do next.

## The seven manuals

| Manual | What it covers |
| --- | --- |
| **Reality** | What actually changed and what it takes before you touch a tool |
| **Shape** | Define the problem, the customer, and whether it's worth building |
| **Build** | Go from spec to a live, working product — web or mobile |
| **Launch** | Get it in front of real people, measure what happens, iterate fast |
| **Monetize** | Start charging, convert free users, make revenue repeatable |
| **Grow** | Build the channels that bring in more of the right users |
| **Operate** | Build the systems that keep the business running as it scales |

## You don't have to read it in order

If you're stuck on converting free users, start at Monetize. If you're figuring out whether your idea is worth building, start at Shape. If the operational layer is falling apart, jump to Operate.

<FieldNote title="One exception">
If you're building something for the first time, read Reality and Shape before you open a builder or write a line of code.
</FieldNote>

## How each chapter is structured

Every chapter follows the same format:

- A clear goal: what you walk away able to do
- Context: why this matters and what most people get wrong (~30%)
- How to do it: step by step, specific, actionable (~70%)
- A checklist: things to complete before moving on
- A prompt: something you can run through an AI tool to speed up the work

## Where Mantle fits in

Mantle is a CRM and billing platform built for app builders. Where relevant, chapters reference Mantle features — usage events, Flows, segments, hosted billing — that make specific operational steps faster. Always optional. Always called out clearly.

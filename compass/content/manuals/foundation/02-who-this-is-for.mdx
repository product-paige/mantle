This guide is for founders building software products, web apps, mobile apps, or both, who want to turn what they've built into a real business.

That covers more ground than you might expect. The person using Lovable to build their first SaaS and the senior developer using Cursor to ship a side project have more in common than they'd think. Both can build. Both need help with everything that comes after the build.

## The builder lifecycle

Most people building software products go through four stages, regardless of their technical background. Knowing where you are changes what you should be doing.

| Stage | What you're doing | Without a system | With a system |
| --- | --- | --- | --- |
| **Builder** | Shipping an MVP, figuring out what's possible | Build something nobody needs because you never talked to real users | Validate before you open a builder |
| **Discoverer** | Getting first users, figuring out who the product is actually for | Optimize the product instead of having conversations | Get out of the builder, talk to people |
| **Monetizer** | Introducing pricing, converting free users | Wait until you "have enough users" to charge. You never will. | Charge from day one with a real system |
| **Operator** | Running retention, analytics, support, billing | Keep building features when the business needs systems | Build the operational layer before it breaks |

<Callout>
The greatest friction shows up at Monetizer and Operator. Not because founders aren't capable, because nothing in the current ecosystem was built to help them through it.
</Callout>

That's the gap this guide fills.

## Who gets the most out of this

- **Non-technical founders using AI builders** — Lovable, Base44, Bolt, Framer. You can ship fast but you're navigating the business and operational layer for the first time.
- **Technical founders and developers using AI tools** — Cursor, GitHub Copilot, Claude Code. You know how to build but you're doing the founder thing for the first time and need the business side laid out.
- **Product-minded operators** — PMs, designers, marketers who can now generate real functionality and need help running what they ship.
- **Micro-SaaS experimenters** — launching multiple products to test demand, tired of rebuilding the same operational infrastructure from scratch every time.

What all four have in common: they can build. What they need help with is everything after.

## A note on mobile

This guide covers both web and mobile app development. Most chapters apply to both. Where mobile has meaningfully different considerations, App Store submission, push notifications, beta testing, platform rules, those are called out specifically. The Build manual has dedicated chapters for the web and mobile paths.
